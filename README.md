# 中国樱花花期预测

基于物候学与最新天气预报的全国赏樱预测网站，覆盖 49 个代表性城市。  
在线访问：[https://rogerzhu.github.io/yinghua-map/](https://rogerzhu.github.io/yinghua-map/)

---

## 项目简介

本网站**完全由 AI 根据用户提供的学术论文、需搜集的资料与目标网页模板自主生成**：从阅读生态学报论文中的樱花花期预报方法，到选定中国 49 个赏樱城市与地点、参考日本樱花预测站点的交互与视觉风格，再到接入天气预报数据并实现地图与表格展示，均由 AI 在对话中完成设计与实现。

**预测结果会随最新天气预报持续订正**：在获取当日或近期天气预报（如 Open-Meteo、中央气象台等）后，按论文中的气温、降水与冬季低温对始花期与花期长度的影响，对各地初绽日、盛开日做小幅订正并更新数据，再推送到本仓库，实现「根据最新天气更新预测」的流程。

---

## 功能概览

- **全国赏樱地图**：49 个城市标注于中国地图，按花况（已开花 / 盛开中 / 即将开放 / 已落花）着色，点击可查看地点、品种与初绽 / 盛开 / 落花日期
- **花期预测表**：按西南、华南、长江中下游、华东、华北、东北、西北分区展示，支持筛选与表格内跳转定位到地图
- **预测方法说明**：简述气温、降水、冬季低温对花期的影响，对应论文中的物候学依据
- **动态「当天」**：花况与页面「更新时间」均按访问当日计算与显示，便于与实时赏樱安排对应

## 技术栈

- 纯静态前端：HTML + CSS + JavaScript
- 地图：[Leaflet](https://leafletjs.com/) + CartoDB 底图
- 无后端依赖，可部署于 GitHub Pages 等静态托管

## 数据与依据

- **物候与预报方法**：谭静、陈正洪、肖玫，《武汉大学樱花花期长度特征及预报方法》，生态学报 2021；南京林业大学樱花研究中心《中国樱花预报》系列
- **城市与地点**：国内主要赏樱城市及代表性景点（如武汉大学、东湖樱园、鸡鸣寺–玄武湖、玉渊潭等），共 49 处
- **天气订正**：使用 [Open-Meteo](https://open-meteo.com/) 及公开气象信息，按 2 月与 3 月初气温、降水对始花期与盛花期做订正（如 2 月偏暖则略提前，3 月初偏冷则略推迟）

## 本地运行与部署

```bash
# 克隆仓库
git clone https://github.com/rogerzhu/yinghua-map.git
cd yinghua-map

# 本地预览（任选其一）
npx serve .
# 或
python3 -m http.server 8080
```

浏览器打开 `http://localhost:8080`（或 serve 提示的地址）即可。

部署到 GitHub Pages：在仓库 Settings → Pages 中选择从 `main` 分支的根目录部署即可，无需构建步骤。

## 关于本项目

> 完全由 AI 根据我提供的论文、应该搜集的资料、想要效果的网页模板生成网站；并且 AI 会根据最新的天气预测更新最新的预测结果。

这段话准确概括了本项目的诞生方式：**论文提供科学依据，资料与模板限定范围与风格，AI 负责从零到可上线产品的实现与迭代**。  

对我而言，这个项目最有价值的地方在于：把一篇关于「樱花花期长度与预报」的论文，变成每个人都能打开、能按城市和日期规划的实用工具；同时把「根据最新天气订正」这件事落进真实的数据与推送流程里，而不是停留在概念。学术结论、开放数据与可复现的更新方式结合在一起，才能长期对「什么时候去哪里看樱花」有用。
